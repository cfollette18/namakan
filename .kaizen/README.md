# .kaizen — Kaizen Agent Configuration

Namakan AI agent setup for Claude Code / OpenClaw agentic development.

## Structure

| Directory | Purpose |
|----------|---------|
| `rules/` | Operational rules and guidelines |
| `commands/` | Custom slash commands |
| `training/` | Agent training data and examples |
| `image-prompts/` | Image generation prompts for website/landing page graphics |
| `design.md` | Design system and visual guidelines |
| `settings.json` | Agent configuration settings |
| `forbidden` | Things agents should not do |
| `CLAUDE.md` | Agent persona and context for this repo |

## Key Files

- **`CLAUDE.md`** — Agent persona, what Namakan builds, tech stack, repo structure
- **`design.md`** — Brand guidelines: colors, typography, visual style
- **`image-prompts/README.md`** — Documentation for image generation prompts
- **`forbidden`** — Safety constraints (no external posts without approval, etc.)

## Quick Start

When working in this repo as a Kaizen agent:
1. Read `.kaizen/CLAUDE.md` for context on Namakan
2. Check `.kaizen/rules/` for operational guidelines
3. Check `image-prompts/` if creating website graphics
